Overview

This presentation outlines the getting, cleaning and exploratory analyses phases of the project. The datasets is provided by Johns Hopkins University in partnership with SwiftKey. The goal of this project is to be able to build a Shiny app containing a text field that predicts the next word of the word or phrase you enter.

The dataset is from a corpus called HC Corpora, in which most of it have already been pre-processed.

Quanteda

Cleaning data involves removing punctuations, special symbols, profanity filtering and more. To ease on this process, we have downloaded and installed a library called Quanteda, which is useful for text analyses.

Operations:

  • Corpus Creation
  • Tokenization
  • Document-feature matrix

Text Analyses

Documentations can be found here.

  1. Corpus
    • Use *Quanteda’s* corpus() function.
    • You may group the data according to the data source i.e. blogs, news, twitter.
  2. Tokenization
    • Use the tokens() function.
    • Remove punctuations, URLs, and other special symbols.
    • Transform to tokens to lower case.

Text Analyses (cont.)

    • Exclude the stopwords from the tokens using the tokens_select() function.
    • Create n-grams using the tokens_ngrams() function. In this case, we created 2 and 3-grams.
  1. Document-feature matrices
    • Use the dfm() function. You can specify the minimum number of characters using the min_char parameter.
    • Create DFMs for each of the tokens.
    • Generate statistics using the textstat_frequency() function with the DFM as the input.

Results

We are able to find out the most common words in the corpus. Note that the data is subset so that the numbers are off by approximately one thousand of the total corpus.

Observations:

  • For single words, we can find mostly pronouns, conjunctions and other parts of speech.
  • For news data, since it is mostly reporting, we hear a lot of the words like “said” and other prepositions.
  • The 2-grams and 3-grams mostly describe events and some politicians.

Prediction Algorithm

Note that the DFMs have already been generated prior to prediction.

  1. Read text input. Get the number of words (\( w \)) in the text field.
  2. Check the value of \( w \).
    1) Let
    \[ n = \begin{cases} 2 & w=1 \\ 3 & w=2 \\ 4 & w\geq 3 \end{cases} \] 2) If \( w = 0 \), wait for input.
    3) Else if \( w > 0 \), find the top 3 \( n \)-grams, that begin with the string according to frequency.
    4) Display the top 3 \( n \)-grams and wait for user selection.
  3. If no \( n \)-grams are found, select the last word in the text input and jump to 2.
